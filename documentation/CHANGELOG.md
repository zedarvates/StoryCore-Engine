## Migration Guide

### Upgrading from 1.5.0 to 2.0.0

#### Breaking Changes
- **Database Schema**: Database schema has changed significantly
- **API Endpoints**: Some API endpoints have changed
- **Authentication**: Authentication system has been completely rewritten
- **User Interface**: User interface has been completely redesigned
- **Configuration**: Configuration format has changed

#### Migration Steps
1. **Backup your existing data**
   ```bash
   pg_dump -U storycore storycore > backup_1.5.0.sql
   ```

2. **Install StoryCore 2.0.0**
   ```bash
   npm install storycore@2.0.0
   ```

3. **Run the migration script**
   ```bash
   storycore migrate
   ```

4. **Update your configuration**
   ```yaml
   # Update config.yaml
   database:
     host: "localhost"
     port: 5432
     name: "storycore"
     user: "storycore"
     password: "your_password"
   
   authentication:
     jwt_secret: "your_new_jwt_secret"
     jwt_expires_in: "1h"
     refresh_token_expires_in: "30d"
   ```

5. **Update your API client**
   ```javascript
   // Old API client
   const response = await fetch('/api/v1/projects', {
     headers: {
       'Authorization': `Bearer ${token}`
     }
   });

   // New API client
   const response = await fetch('/api/v2/projects', {
     headers: {
       'Authorization': `Bearer ${token}`,
       'Content-Type': 'application/json'
     }
   });
   ```

#### Configuration Changes
- **Database**: Connection parameters have changed
- **Authentication**: JWT configuration has changed
- **Security**: Security settings have changed
- **Performance**: Performance settings have changed
- **Monitoring**: Monitoring settings have changed

#### API Changes
- **Versioning**: API has been versioned (v1 to v2)
- **Authentication**: Authentication headers have changed
- **Endpoints**: Some endpoints have changed
- **Responses**: Response format has changed
- **Error Handling**: Error handling has improved
