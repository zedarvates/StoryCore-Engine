feat: Amélioration du sélecteur de fichiers pour la version web

Implémente une architecture progressive à 3 niveaux pour le dialogue
"Open Existing Project" afin d'offrir la meilleure expérience utilisateur
possible selon l'environnement.

## Problème résolu
La version web utilisait uniquement un modal personnalisé (FolderNavigationModal)
au lieu d'utiliser les dialogues natifs disponibles dans les navigateurs modernes,
créant une expérience incohérente avec la version Electron.

## Solution implémentée

### Architecture à 3 niveaux
1. **Electron** : Dialogue natif OS (Windows Explorer, macOS Finder)
   - Expérience optimale (⭐⭐⭐⭐⭐)
   - Aucun changement

2. **Navigateurs modernes** : File System Access API (showDirectoryPicker)
   - Chrome 86+, Edge 86+, Opera 72+
   - Dialogue natif du navigateur
   - Expérience améliorée de ⭐⭐ à ⭐⭐⭐⭐

3. **Navigateurs anciens** : Modal personnalisé (fallback)
   - Firefox, Safari (versions actuelles)
   - Expérience acceptable (⭐⭐⭐)
   - En attente du support de l'API
