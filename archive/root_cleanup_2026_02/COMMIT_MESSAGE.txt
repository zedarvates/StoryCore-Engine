fix(i18n): Résolution de l'erreur "useI18n must be used within an I18nProvider"

## Problème
Le composant MenuBar utilisait le hook useI18n() sans que l'application
soit enveloppée dans le provider I18nProvider, causant un crash au démarrage.

## Solution
- Ajout de l'import I18nProvider dans App.tsx
- Enveloppement de l'application avec I18nProvider
- Configuration avec defaultLanguage="en" et enableAutoDetect={false}

## Changements
- creative-studio-ui/src/App.tsx
  * Import ajouté: I18nProvider from '@/utils/i18n'
  * Provider ajouté dans la hiérarchie des contextes
  * Configuration: langue anglaise par défaut, pas de détection auto

## Hiérarchie des Providers
ErrorBoundary
  └─ I18nProvider (NOUVEAU)
      └─ LanguageProvider
          └─ NavigationProvider
              └─ SecretModeProvider
                  └─ LLMProvider
                      └─ AppContent
