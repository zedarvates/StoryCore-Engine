**Version:** 1.0  
**Last Updated:** 2024

## Quick Reference Card

This document provides a comprehensive reference for all keyboard shortcuts available in the StoryCore Creative Studio. Shortcuts are organized by menu category for easy lookup.

### Platform Conventions

- **Windows/Linux:** Uses `Ctrl` key
- **macOS:** Uses `⌘` (Command) key

Throughout this document:
- `Ctrl` means `Ctrl` on Windows/Linux and `⌘` on macOS
- `Alt` means `Alt` on Windows/Linux and `⌥` (Option) on macOS
- `Shift` means `⇧` on all platforms

## File Menu Shortcuts

### Project Management

| Action | Windows/Linux | macOS | Description |
|--------|---------------|-------|-------------|
| **New Project** | `Ctrl+N` | `⌘N` | Create a new StoryCore project with default settings |
| **Open Project** | `Ctrl+O` | `⌘O` | Open an existing project from your file system |
| **Save Project** | `Ctrl+S` | `⌘S` | Save the current project (disabled if no project is loaded) |
| **Save Project As** | `Ctrl+Shift+S` | `⌘⇧S` | Save the current project with a new name or location |

### Export Operations

Export operations do not have keyboard shortcuts. Access them via **File > Export** menu.

### Recent Projects

Recent projects do not have keyboard shortcuts. Access them via **File > Recent Projects** menu to see your 10 most recently opened projects.

## Edit Menu Shortcuts

### Undo/Redo Operations

| Action | Windows/Linux | macOS | Description |
|--------|---------------|-------|-------------|
| **Undo** | `Ctrl+Z` | `⌘Z` | Revert the last action (disabled if undo stack is empty) |
| **Redo** | `Ctrl+Y` | `⌘Y` | Reapply the last undone action (disabled if redo stack is empty) |

### Clipboard Operations

| Action | Windows/Linux | macOS | Description |
|--------|---------------|-------|-------------|
| **Cut** | `Ctrl+X` | `⌘X` | Remove selected content and place in clipboard (requires selection) |
| **Copy** | `Ctrl+C` | `⌘C` | Copy selected content to clipboard (requires selection) |
| **Paste** | `Ctrl+V` | `⌘V` | Insert clipboard content at current position (requires clipboard content) |

### Preferences

| Action | Windows/Linux | macOS | Description |
|--------|---------------|-------|-------------|
| **Preferences** | `Ctrl+,` | `⌘,` | Open user preferences dialog |

## View Menu Shortcuts

### Zoom Controls

| Action | Windows/Linux | macOS | Description |
|--------|---------------|-------|-------------|
| **Zoom In** | `Ctrl+=` | `⌘=` | Increase timeline zoom level by one step (disabled at maximum zoom) |
| **Zoom Out** | `Ctrl+-` | `⌘-` | Decrease timeline zoom level by one step (disabled at minimum zoom) |
| **Reset Zoom** | `Ctrl+0` | `⌘0` | Reset timeline zoom to 100% (disabled if already at 100%) |

### Display Toggles

| Action | Windows/Linux | macOS | Description |
|--------|---------------|-------|-------------|
| **Toggle Fullscreen** | `F11` | `F11` | Enter or exit fullscreen mode |

### Panel Visibility

Timeline, Grid, and Panel visibility toggles do not have keyboard shortcuts. Access them via the **View** menu.

## Project Menu Shortcuts

Project menu items do not have keyboard shortcuts. Access project settings, characters, sequences, and assets via the **Project** menu.

**Note:** Most Project menu items are disabled when no project is loaded.

## Tools Menu Shortcuts

Tools menu items do not have keyboard shortcuts. Access AI tools and utilities via the **Tools** menu:

- **LLM Assistant** - Configure the StoryCore AI assistant
- **ComfyUI Server** - Configure backend AI generation server
- **Script Wizard** - Launch guided script-to-shots workflow
- **Batch Generation** - Process multiple shots in batch mode
- **Quality Analysis** - Run QA Engine on current project

**Note:** Some Tools menu items are disabled when no project is loaded.

## Navigation Shortcuts

### Menu Bar Navigation

| Action | Windows/Linux | macOS | Description |
|--------|---------------|-------|-------------|
| **Focus Menu Bar** | `Alt` | `⌥` | Focus the first menu in the menu bar |
| **Navigate Menus** | `←` `→` | `←` `→` | Move between menus when menu bar is focused |
| **Open Menu** | `Enter` or `↓` | `Enter` or `↓` | Open the focused menu |
| **Close Menu** | `Esc` | `Esc` | Close current menu and return focus to menu bar |

### Menu Item Navigation

| Action | Windows/Linux | macOS | Description |
|--------|---------------|-------|-------------|
| **Navigate Items** | `↑` `↓` | `↑` `↓` | Move between items in an open menu |

| **Open Submenu** | `→` | `→` | Open submenu of focused item (if available) |
| **Close Submenu** | `←` | `←` | Close submenu and return to parent menu |
| **Jump to First** | `Home` | `Home` | Jump to first item in current menu |
| **Jump to Last** | `End` | `End` | Jump to last item in current menu |

## Tips for Power Users

### Workflow Optimization

1. **Quick Save Workflow**
   - Use `Ctrl+S` (⌘S) frequently to save your work
   - Use `Ctrl+Shift+S` (⌘⇧S) to create project snapshots with different names

2. **Efficient Editing**
   - Master `Ctrl+Z` (⌘Z) and `Ctrl+Y` (⌘Y) for quick undo/redo
   - Use clipboard shortcuts (`Ctrl+X/C/V`) for rapid content manipulation

3. **View Management**
   - Use `Ctrl+0` (⌘0) to quickly reset zoom after detailed work
   - Press `F11` to maximize workspace in fullscreen mode

4. **Keyboard-First Navigation**
   - Press `Alt` (⌥) to access menu bar without mouse
   - Use arrow keys to navigate menus efficiently
   - Press `Enter` to activate items, `Esc` to cancel

### Accessibility Features

- **Full Keyboard Navigation:** All menu functions are accessible via keyboard
- **Screen Reader Support:** Menu items announce their state and shortcuts
- **Focus Indicators:** Visual feedback shows which item is focused
- **ARIA Compliance:** Follows WAI-ARIA menubar pattern for assistive technologies

### Customization

Currently, keyboard shortcuts are fixed and cannot be customized. Future versions may include shortcut customization in **Edit > Preferences**.

## Disabled Shortcuts

Some shortcuts are context-sensitive and may be disabled:

| Shortcut | Disabled When |
|----------|---------------|
| `Ctrl+S` (Save) | No project is loaded |
| `Ctrl+Shift+S` (Save As) | No project is loaded |
| `Ctrl+Z` (Undo) | Undo stack is empty |
| `Ctrl+Y` (Redo) | Redo stack is empty |
| `Ctrl+X` (Cut) | No content is selected |
| `Ctrl+C` (Copy) | No content is selected |
| `Ctrl+V` (Paste) | Clipboard is empty |
| `Ctrl+=` (Zoom In) | Already at maximum zoom |
| `Ctrl+-` (Zoom Out) | Already at minimum zoom |
| `Ctrl+0` (Reset Zoom) | Already at 100% zoom |

## Platform-Specific Notes

### macOS

- The `⌘` (Command) key replaces `Ctrl` for most shortcuts
- The `⌥` (Option) key replaces `Alt` for menu bar access
- Function keys (like `F11`) may require pressing `Fn` depending on system settings

### Windows/Linux

- Standard `Ctrl` key is used for all shortcuts
- `Alt` key accesses the menu bar
- Function keys work directly without modifier keys

### Browser Conflicts

Some shortcuts may conflict with browser shortcuts:

- `Ctrl+N` (New Project) may open a new browser window
- `Ctrl+O` (Open Project) may open browser's file dialog
- `Ctrl+S` (Save Project) may trigger browser's save page dialog
- `F11` (Fullscreen) may toggle browser fullscreen

**Solution:** Use the StoryCore Creative Studio as a standalone application or in a dedicated browser window to avoid conflicts.
